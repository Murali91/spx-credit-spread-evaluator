# Guidelines for AI Agent Contributions

This document outlines how autonomous or semi‑autonomous agents (such as OpenAI Codex or other code‑generation tools) should interact with this repository.  The goal is to ensure that all automated contributions adhere to the project’s architecture, style and safety requirements.

## Scope of Work

1. **Follow the Specification** – All changes must conform to the high‑level requirements in `docs/BRD.md` and the detailed decision logic in `docs/Decision_Spec_v0.1.md`.  Do not introduce new business logic or strategy factors without a corresponding update to the specification documents.
2. **Maintain Explainability** – The decision engine must always return a result in the form `DecisionResult(decision="ENTER" or "WAIT", reasons=[...])`.  Each reason should correspond to one of the evaluated factors.  Agents should never remove or obscure explanatory information.
3. **No Brokerage or Trade Execution** – Do not add any functionality that connects to brokerage APIs or executes trades.  This tool is strictly for educational analysis and market timing suggestions.
4. **Free‑First Data Sources** – When fetching market or event data, prefer free sources as defined in `docs/Data_Sources.md`.  Paid APIs should only be used if explicitly allowed by project maintainers.
5. **Guard Against Bias and Overfitting** – When tuning thresholds or adding new logic, rely on data‑driven methods (e.g. historical backtesting) and document all changes.  Avoid over‑optimizing on a specific time period and maintain the educational purpose of the tool.

## Contribution Workflow

- **Small, Self‑Contained PRs** – Each pull request should implement a single feature or fix.  This makes review easier and reduces the chance of introducing bugs.
- **Tests Required** – When adding or modifying code in `engine/` or `data/providers/`, corresponding tests in `tests/` must be added or updated.  PRs without tests will not be accepted.
- **Documentation** – If you change the decision logic or user‑facing behavior, update the relevant documents in `docs/` and provide clear commit messages summarizing the rationale.  Keep docs consistent with the implemented logic.
- **Human Review** – All PRs generated by agents require human approval before merging.  Agents must not override branch protections or merge changes autonomously.

## Code Style

- Use standard Python type hints and keep functions short and focused.
- Avoid external dependencies beyond what is listed in `requirements.txt` unless explicitly instructed by maintainers.
- Format code with a consistent style (e.g. via Black) if a formatter is later adopted, and ensure tests pass before proposing a PR.

By adhering to these rules, AI agents help maintain quality and consistency in the project while accelerating development.
